    Regular expression flags

    If the start delimiter ends with an initial substring of the end delimiter, e.g. in the case of C-style comments, the behaviour differs from the model regex. In this implementation, the end must share no characters with the start, so e.g. /asterisk/ is not considered to be both the start and end of a comment. /asterisk/xy/asterisk/ is considered to be a single comment with contents /xy/.
